# openai-for-open-source-sponsorship-experience

> 本文分享了作者如何通过为 OpenViking 项目（一个面向 Agent 时代的 Context Database）做贡献，成功获得 OpenAI for Open Source 赞助的经历。作者在项目早期加入，主要从用户体验和生态完善角度贡献力量：编写 usage examples、支持 Rust 版本 CLI 和 TUI、参与 skills 调用功能、以及创建 openclaw-eval b

## 概述

本文是 Zayn（Yang Johnson）分享的获得 OpenAI for Open Source 赞助的经历。

## 项目背景

赞助是通过 **OpenViking** 项目（https://github.com/volcengine/openviking）获得的。

OpenViking 是 2025 年 11 月立项的团队项目，目标是成为 Agent 时代的 Context Database。

## 贡献历程

作者在项目早期就加入贡献，主要贡献方向：

1. **编写 Usage Examples** - 提供 Q&A、chat with memory、MCP、skills 等使用示例作为最佳实践
2. **Rust CLI 支持** - 用 AI 生成 + AI review，下班通勤路上 10 分钟完成
3. **Rust TUI 支持** - 目前已被 Web Console 需求替代
4. **Skills 调用支持** - 春节期间完成
5. **openclaw-eval benchmark 方案** - 远程控制电脑完成，目前仍沿用该方案框架

## 申请经验

- 了解到 OpenAI for OpenSource 项目后直接申请，「瞎填」竟通过
- 两个月后收到通过通知

## 核心心得

> 找个认同相关理念的工作，不必要去改最核心的内核，从生态和用户易用性上补充必要的、自己认为有价值的工作。

## 彩蛋

当成为 top contributors 时，可以直接找 maintainer 接手相关 feature 的 review 和批量合入。
