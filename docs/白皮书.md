## 一、项目背景介绍

### 1.1 LLM Wiki 模式的起源

2026 年 4 月，AI 大神 Andrej Karpathy 发推提出 **LLM Wiki 模式**——让大语言模型自己管理和组织知识库，而不是仅仅把 AI 当作搜索工具。

**核心理念：不是你在管理知识库，是知识库自己在成长。**

### 1.2 本项目的起源

受到 Karpathy 理念的启发，以 Obsidian 为本地知识库核心，以 OpenClaw 为 AI Agent 执行引擎，实现全自动化的知识摄入与运营。

### 1.3 解决的问题

- ⏰ **时间浪费**：读文章 30 分钟，整理笔记又要 30 分钟 → AI 自动完成
- 🔍 **找不到知识**：积累了几百篇笔记，关键时刻想不起来 → 智能检索
- 📦 **知识孤岛**：飞书/微信/网页分散各处 → 全渠道入库
- 🧠 **资产闲置**：沉淀了大量知识，无法转化为可复用数字资产 → 自动输出

## 二、方案概述

### 2.1 三层架构

| 层级 | 目录 | 内容 | 负责主体 |
|------|------|------|---------|
| Raw Materials | `raw/` | 原始资料：PDF、文章、截图 | 人类（主要） |
| Wiki Layer | `wiki/` | AI 生成：摘要、概念、索引 | OpenClaw Agent |
| Schema Layer | `CLAUDE.md` | 规范、规则、运营策略 | 人类 + AI 共同维护 |

### 2.2 技术栈

**核心工具**：Obsidian · OpenClaw · QMD · GitHub

**自研脚本**：auto_ingest.py · ingest_inbound.py · lint_wiki.py · audit_review.py

**自动化**：raw/ 监控（每15分钟）· Auto-Ingest（周1/3/5）· Weekly Lint（每周日）

### 2.3 工作流

```
摄入(Ingest) → 整理(Compile) → 检索(Query) → 检查(Lint) → 输出(Output)
```

- **摄入**：飞书文件 / 飞书云文档 / 任意 URL / Web Clipper
- **整理**：TF-IDF 自动生成 summary，自动写入 wiki/
- **检索**：QMD 语义搜索 + Dataview 索引查询
- **检查**：lint_wiki.py 定期健康检查
- **输出**：D3.js 图谱 + python-pptx 自动 PPT

### 2.4 与图里方案对比

| 环节 | 图里方案 | 咱们方案 |
|------|---------|---------|
| 摄入 | Clipper → raw/ | Clipper → Clippings → watcher → auto_ingest ✅ |
| 整理 | 手动整理 | TF-IDF 自动生成 summary ✅ |
| 检索 | Dataview | Dataview + QMD 语义搜索（双轨）✅ |
| 图谱 | Obsidian 原生 | D3.js 自定义（搜索/高亮/跳转）✅ |
| 输出 | Marp（手动） | python-pptx 自动生成 PPT ✅ |
| 运营 | 无 | Cron 自动 + Lint 健康检查 ✅ |

## 三、使用指南

### 3.1 快速启动（30 分钟）

1. 安装 Obsidian，克隆 GitHub 仓库
2. 安装插件：Dataview、Marp、Obsidian Git
3. 配置 OpenClaw，飞书连接
4. 测试入库：发一个链接给机器人
5. 查询知识：用 Dataview 或直接问小桃子

### 3.2 内容入库（4 种方式）

- **飞书文件**：直接发送文件给小桃子机器人
- **飞书云文档**：发送云文档链接（支持 docx/wiki/sheets/bitable）
- **任意 URL**：发送网页链接，自动抓取正文
- **Web Clipper**：保存到 `Clippings/`，watcher 自动检测入库

### 3.3 知识检索

- **Dataview**：在 Obsidian 中用 DataviewQL 查询
- **直接问小桃子**：在飞书提问，AI 基于知识库回答
- **QMD 语义搜索**：自然语言检索（Gateway 重启后生效）

### 3.4 图谱使用

打开 `output/wiki-graph.html`，D3.js 力导向图支持：
- 🔍 搜索节点名称，高亮相关链路
- 🖱️ 拖拽节点，调整布局
- 📖 双击节点，跳转到 Obsidian 对应页面

### 3.5 健康检查

```bash
cd ~/Documents/Obsidian\ Vault
python3 lint_wiki.py .
```
